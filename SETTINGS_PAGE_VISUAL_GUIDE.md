## Interactions

### Theme Switch
```
User Action:  Toggle switch
Immediate:    Theme changes
Feedback:     "? Theme changed to [Mode]"
Persistence:  Saved to SecureStorage
Duration:     Message auto-clears (2s)
```

### Clear Cache
```
User Action:Click "Clear Cache"
Immediate:    Button disabled, spinner shows
Process:      Cache clearing operation
Feedback:     "? Cache cleared successfully"
Update:       Cache status shows "No cached data"
Duration:     Message auto-clears (2s)
Re-enable:  Button becomes clickable again
```

### Refresh Cache Status
```
User Action:  Click refresh button (?)
Immediate:    Button becomes spinner (?)
Process:      Check cache timestamps
Update:Status label updates
Feedback:     "? Cache status refreshed"
Duration:     Message auto-clears (1.5s)
```

## Example Flows

### First Time User
1. Opens Settings
2. Sees default Light theme
3. Toggles to Dark mode
4. Theme changes immediately
5. Preference saved
6. Returns to Settings later
7. Dark mode still active ?

### Cache Management
1. Opens Settings
2. Sees "Cache active (5.2h old)"
3. Clicks Clear Cache
4. Button disables, spinner shows
5. "Clearing cache..." message
6. "? Cache cleared successfully"
7. Status shows "No cached data"
8. App will reload data when needed

### Theme Persistence
1. User sets Dark mode
2. Closes app
3. Reopens app
4. App starts in Dark mode ?
5. All pages respect theme
